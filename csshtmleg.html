This text is red in color. Note that it is a good practice to end inline style declarations with a ; .

Note that it's important to have both opening and closing curly braces ({ and }) around each element's style rule(s). You also need to make sure that your element's style definition is between the opening and closing style tags. Finally, be sure to add a semicolon to the end of each of your element's style rules.

Please note elements are written as it is in style rules. that means you can write style rules for all h2 or p or ul elements. But suppose if you want to only specifically write a style rule for a particular thing then you need to add an id to identify it. A class is ideal if a group of things should have same style rule. For example if you would like a particular background color for certain blocks then simply use the class name and it will automatically include style rules for the particular class.

Classes are reusable styles that can be added to HTML elements.
